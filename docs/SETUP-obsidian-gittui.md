# AgentOps Vault — Setup Guide

**Part of the obsidian-gittui plan** — see [PLAN-obsidian-gittui.md](./PLAN-obsidian-gittui.md).

---

## Prerequisites

- Obsidian 1.12.7+ with Command Line Interface enabled
  (Settings → General → Command line interface → toggle on, restart terminal)
- Git
- Node.js 18+ (for Pi extensions)
- Pi coding agent with extension support

---

## Quick start

```sh
# 1. Bootstrap the vault (idempotent)
./scripts/setup-agentops.sh

# 2. Verify
ao status

# 3. Open in Obsidian
ao open
```

Then in Obsidian, install community plugins (see below).

---

## What gets set up

```
~/repos/AgentOps/
  Inbox/              # Quick captures, uncategorized
  Projects/           # Specs, plans, designs, architecture
  Runs/               # Per-agent-run session notes
  Reviews/            # Review batches, approval summaries
  System/
    Templates/        # Run.md, Spec.md, Review.md
    Dashboards/       # Dataview dashboards: AgentOps, Runs, Reviews
    plugin-guard.md   # Documented required plugin list
    plugin-guard-last-run.md  # Auto-generated by ao repair

  .gitignore          # Partial git tracking (config, not cache/workspace)
  .obsidian/
    core-plugins.json
    community-plugins.json
    app.json
    appearance.json
```

## Step-by-step

### 1. Run the setup script

```sh
cd ~/dotfiles
./scripts/setup-agentops.sh
```

This does everything idempotently:

- Creates the vault skeleton if absent
- Seeds `.obsidian` plugin config
- Seeds templates and dashboards
- Initializes git tracking
- Symlinks `ao` → `~/.local/bin/ao`
- Runs plugin guard repair
- Expands `$HOME` in config templates

### 2. Symlink ao manually (if setup skipped)

```sh
ln -sf ~/dotfiles/scripts/ao ~/.local/bin/ao
hash -r  # zsh: rehash
```

### 3. Install community plugins

```sh
ao open
```

In Obsidian:

1. Settings → Community plugins → Turn off Safe mode
2. Browse — install these plugins (click each, Install + Enable):

| Plugin | ID | Purpose |
|--------|----|---------|
| Dataview | `dataview` | Query engine for dashboards |
| Tasks | `obsidian-tasks-plugin` | Task management |
| Templater | `templater-obsidian` | Template insertion |
| Periodic Notes | `periodic-notes` | Daily/weekly notes |
| Calendar | `calendar` | Calendar view |
| Obsidian Git | `obsidian-git` | Auto-commit + sync |
| Commander | `cmdr` | Custom toolbar |
| Buttons | `buttons` | Clickable actions |
| Meta Bind | `obsidian-meta-bind-plugin` | Inline metadata controls |
| Advanced URI | `obsidian-advanced-uri` | URI actions |
| Quick Add | `quickadd` | Quick capture |
| Kanban | `obsidian-kanban` | Kanban boards |
| Excalidraw | `obsidian-excalidraw-plugin` | Diagrams |
| Omnisearch | `omnisearch` | Full-text search |
| Linter | `obsidian-linter` | Formatting |
| Style Settings | `obsidian-style-settings` | CSS tweaks |

After installing, run:

```sh
ao repair
```

This writes a guard report confirming all plugin folders exist.

### 4. Update agent-workflow config

If you have an existing `~/.config/agent-workflow/config.json`, update the vault path:

```sh
# Check current value
grep obsidianVault ~/.config/agent-workflow/config.json

# If it still says ~/repos/Obsidian, update it:
python3 -c "
import json
p = '$HOME/.config/agent-workflow/config.json'
d = json.load(open(p))
d['obsidianVault'] = '~/repos/AgentOps'
d['obsidianFolders'] = {
    'spec': 'Projects',
    'design': 'Projects',
    'arch': 'Projects',
    'plan': 'Projects',
    'pr': 'Reviews',
    'review': 'Reviews',
    'note': 'Inbox',
}
json.dump(d, open(p, 'w'), indent=2)
"
```

Or delete the user config to inherit the dotfiles default:

```sh
rm ~/.config/agent-workflow/config.json
./scripts/bootstrap-dirs.sh  # re-materializes from default
```

### 5. Verify end-to-end

```sh
# Shell helper
ao status

# Pi commands — from inside a Pi session:
#   /obsidian-note spec "test agentops setup"
#   /cp
#   /review-gate
#   /review-list
```

Check:

- Note lands in `~/repos/AgentOps/Projects/`, not `~/repos/Obsidian`
- Plugin guard report exists at `System/plugin-guard-last-run.md`
- No accidental writes to the old main vault

## Environment variables

| Variable | Default | Purpose |
|----------|---------|---------|
| `AGENTOPS_VAULT` | `$HOME/repos/AgentOps` | Override vault path |
| `PI_OBSIDIAN_VAULT` | (fallback) | Same, alternative name |
| `AGENTOPS_ALLOW_OLD_VAULT` | unset | Set to `1` to allow writes to `~/repos/Obsidian` |
| `AO_OBSIDIAN_CLI` | `obsidian` | Override Obsidian CLI binary |

Resolution order (highest first):

1. `AGENTOPS_VAULT` env var
2. `PI_OBSIDIAN_VAULT` env var
3. `params.vault` tool/command argument
4. repo-local `.agent-workflow.local.json` / `.agent-workflow.json`
5. User `~/.config/agent-workflow/config.json`
6. Dotfiles default (`~/repos/AgentOps`)

## Git tracking

The AgentOps vault uses **partial git tracking** — only configuration, templates,
and markdown notes are committed. Workspace state, plugin binaries, and cache
are gitignored.

```yaml
Tracked:
  .obsidian/app.json
  .obsidian/appearance.json
  .obsidian/core-plugins.json
  .obsidian/community-plugins.json
  .obsidian/plugins/*/data.json (where useful)
  System/Templates/
  System/Dashboards/
  System/plugin-guard.md
  System/plugin-guard-last-run.md
  Inbox/
  Projects/
  Runs/
  Reviews/

Ignored:
  .obsidian/workspace*.json
  .obsidian/cache/
  .obsidian/plugins/*/main.js
  .obsidian/plugins/*/styles.css
  .trash/
  Reviews/_artifacts/raw/
```

See `.gitignore` in the vault root for the full list.

## Troubleshooting

### "Refusing to write to old main vault"

```text
Error: Refusing to write to old main vault (/Users/me/repos/Obsidian).
       AgentOps vault should be at /Users/me/repos/AgentOps.
       Set AGENTOPS_VAULT env var or update agent-workflow config.
       To force: set AGENTOPS_ALLOW_OLD_VAULT=1
```

**Fix:** Update your `obsidianVault` config as shown in step 4 above, or set:

```sh
export AGENTOPS_VAULT="$HOME/repos/AgentOps"
```

### "Obsidian CLI not found"

Obsidian CLI must be enabled in Obsidian settings. On macOS, the CLI binary is
typically at `/Applications/Obsidian.app/Contents/Resources/obsidian-cli`.
Enable it from Obsidian → Settings → General → Command line interface → toggle on,
then restart your terminal.

### "Plugin folders missing"

You installed community plugins in Obsidian but the folders don't exist yet.
This is normal if the vault hasn't been opened in Obsidian yet. Run:

```sh
ao open    # first open triggers plugin download
# wait for plugins to install
ao repair  # verify
```

### "Review batch not found"

The `.review-gate/` directory lives inside the repo you're working in, not in
the AgentOps vault. Make sure you run `/review` from inside your project repo,
and that a batch has been created first with `/review-batch <sandbox-path>`.

## Commands reference

### Shell

```sh
ao                    # default: open
ao open               # open AgentOps vault in Obsidian
ao status             # show vault state
ao repair             # run plugin guard + auto-fix
ao note <text>        # quick capture to Inbox
ao help               # show full usage
```

### Pi (inside agent session)

```sh
/obsidian-note <kind> <title>     # create/update spec/plan/review/note
/obsidian-open <path>             # open a note
/obsidian-update <path>           # refresh existing note with session context
/cp                                # open control plane cockpit
/review                            # open review overlay for current batch
/review-gate                       # toggle review gate on/off
/review-batch <path> [agent]      # create batch from sandbox diff
/review-list                       # list all batches
/review-sandbox <branch>          # create git worktree sandbox
```
